# Duecare Gemma Chat (Kaggle Notebook 3 of 3)

A pure Gemma 4 chat playground. **NOT the safety harness** — no
moderation pipeline, no audit trail, no slideshow. Just a clean chat
UI bound to FastModel for any Gemma 4 variant (default 31B-it on
T4 ×2). Multimodal-capable (image upload). Cloudflared tunnel like
the live demo.

## Wheels included (3)

`duecare-llm-core`, `duecare-llm-models`, `duecare-llm-chat`.

The `duecare-llm-chat` package contains a minimal FastAPI app + dark-
themed chat HTML page. NO Duecare pipeline, NO benchmark, NO evidence.

## What the chat UI exposes

- Multi-turn chat with full message history (client-side state)
- Image upload (`store://<id>` references resolved server-side to
  base64 data URIs before forwarding to Gemma)
- Generation params: `max_new_tokens`, `temperature`, `top_p`, `top_k`
- Model badge polling `/api/model-info` every 30 s
- Code block formatting in responses

## Publishing options

### A. Paste-into-Kaggle (preferred)

1. Open https://www.kaggle.com/code/taylorsamarel/duecare-gemma-chat
   (create new kernel if it doesn't exist yet).
2. Replace the single code cell with [`kernel.py`](./kernel.py).
3. Confirm side panel: GPU T4 ×2 · Internet ON · the four Gemma 4
   model attachments · `taylorsamarel/duecare-gemma-chat-wheels`
   dataset attached.
4. Save & Run All. The cloudflared URL appears within ~30 s (E4B) or
   ~3 min (31B).

### B. Script-driven push (when explicitly approved)

```bash
python scripts/push_kaggle_demo.py --kernel gemma-chat --enable-gpu false
```

## Model variant selection

The kernel defaults to `GEMMA_MODEL_VARIANT = "31b-it"`. Edit the
constant near the top of `kernel.py` to switch to E4B (faster cold
start) or E2B (smallest, on-device-friendly).
