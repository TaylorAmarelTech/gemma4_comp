How it works (per request)

Every input goes through the same 5-stage pipeline. Each stage emits a trace step you can watch live on the Workspace and Examples pages.

1
Heuristic prescan
11 keyword signals scored 0–10 (passport-confiscation, excessive fee, debt-bondage, …). Establishes a severity floor.
≈ 5 ms
→
2
GREP knowledge base
Regex match against 27 statutes / conventions / ILO indicators (RA 8042, ILO C181, Palermo Art 3, …).
≈ 20 ms
→
3
RAG retrieval
sentence-transformers (MiniLM) → cosine top-K. Surfaces semantically-relevant passages the regex missed.
≈ 200 ms
→
4
Tool calls
Deterministic lookup_statute(juris, topic) and lookup_hotline(country) chosen from prescan signals + locale.
≈ 5 ms
→
5
Gemma 4 classify
Severity-anchored prompt + heuristic findings + KB hits as context. Returns BLOCK / REVIEW / PASS with reasoning.
≈ 8–18 s (T4 4-bit)
