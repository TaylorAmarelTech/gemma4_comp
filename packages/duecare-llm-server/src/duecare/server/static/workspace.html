Workspace — bring your own data

Paste text, upload a file, drop an archive of documents, or import a JSONL of cases. Everything routes through the same traced queue you saw on the Demo page.

Free-form text moderation

Paste any recruitment ad, chat message, contract excerpt, or complaint letter. Pick the kind of analysis you want and the worker locale (used for the hotline lookup).

Single file upload (image / PDF / DOCX / TXT)

OCR (pytesseract) for images. PDF text extraction via pypdfium2. Docx via stdlib zipfile + XML.

Archive upload (.zip / .tar / .tar.gz)

Drop an archive of documents. Every file inside gets queued as its own task. Caps: 500 files, 25 MB/file, 500 MB total. Supported types: txt, md, csv, pdf, docx, png, jpg, jpeg, tiff, bmp, gif, webp.

Batch results

JSONL batch import

Paste one JSON object per line. Each must have a text field and optionally a locale. Every row gets queued as its own moderation task. Useful for triaging exported platform reports, scraped postings, or chat logs.

parses up to 200 lines / submission

Prompt templates

Click a template to copy it into the Single text tab. These are starting points the operator can edit before running.

Generate complaint / cause-of-action letter

Pick a jurisdiction + violation type, fill in the operator-known facts, and the system renders a draft letter / sworn affidavit / takedown request you can submit to the relevant authority. Templates currently cover PH (DMW, POLO), HK (PCPD, money lenders, demands), Indonesia (BP2MI), Nepal (FET), GCC states (WPS), US (TVPA + FLSA), EU (GDPR), and Facebook (platform takedown).

After a moderation run on the other tabs, the verdict will include a suggested_complaint_template based on the patterns detected; pick that suggestion or any other template here.
