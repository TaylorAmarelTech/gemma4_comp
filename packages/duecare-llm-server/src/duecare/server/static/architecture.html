How the moderation pipeline works (deep dive)

A reading-grade explanation of every stage. Each section answers what it does, why it's there, what it can / can't catch, and cost in time + memory. Use this to brief judges or NGO partners.

1

Heuristic prescan

Eleven keyword signals organised into 3 tiers (critical · strong · legitimate-context). The signal set is tuned to the rubric vocabulary that matters for ILO C029 forced-labour indicators and PH RA 8042 / DMW recruitment violations. Every signal has a numeric weight; the sum (capped 0–10) becomes the heuristic severity floor.

Why a heuristic at all?

It runs in under 5 ms, no GPU, no network. That gives Gemma a deterministic anchor to reason from — Gemma sees the prescan findings in its prompt, so it can say "the heuristic flagged X; confirm or refute" rather than free-associating. Also: if Gemma underrates a clearly-severe post (severity 1/10 on something with passport confiscation), the heuristic floor kicks in via final_sev = max(gemma, heuristic).

What it catches

  • Critical (verdict-driving): passport_confiscation, excessive_fee_with_currency, movement_restriction, debt_bondage, coercion_authority
  • Strong (escalating): identity_doc_request, placement_fee_mention, private_channel_payment, excessive_overtime
  • Legitimate (de-escalating): poea_license_mention, employer_pays_principle

What it misses

Polite or euphemistic language ("we hold the passport for safekeeping") may not match a literal keyword. Stages 2 and 3 catch those via the KB.

2

GREP knowledge base

Regex match of the input's tokens + a curated phrase set ("ILO C029", "RA 8042", "kafala", …) against 27 KB passages: 8 statutes, 7 ILO conventions, 4 ILO forced-labour indicators, plus 13 hotline entries. Each passage has tags + jurisdiction + topic so we can score relevance without an embedding model.

Why GREP before RAG?

Two reasons: (a) it always works, even if the embedder didn't load; (b) some matches you actively want to be explainable ("we returned ph_ra8042_sec6a because the input contained the literal phrase 'placement fee'"). RAG returns top-K by cosine, which can drift on weird inputs.

What it catches

Anything where the user uses the same vocabulary as the KB passages — statute names, convention numbers, well-known terms like "kafala" or "POLO".

What it misses

Paraphrases ("they keep my documents for safekeeping") match the KB's passport tag only via tokens, so the score is lower. Stage 3 fixes this.

3

RAG retrieval

Encodes the input via sentence-transformers/all-MiniLM-L6-v2 (22 MB model, runs on CPU in ~200 ms). KB passages are pre-embedded once at startup. Cosine top-K on the in-memory matrix returns the most-similar passages, regardless of whether the input shares any literal tokens.

Why RAG when we have GREP?

Coverage. A worker writing in their own language, or paraphrasing the way real complaints actually sound, will rarely use the KB's exact words. Embedding-based retrieval catches "they took my papers" → ILO C029 indicator #7. Surfacing both stages' hits to Gemma is what the literature calls retrieval-augmented generation (RAG).

What it catches

Semantic matches: euphemisms, near-synonyms, paraphrases. Multilingual to a degree — MiniLM was trained on parallel corpora that include Tagalog, Indonesian, Hindi.

What it misses

Adversarial inputs that share semantic surface with the KB but mean the opposite (e.g. "we DO NOT hold passports"). Stage 5 (Gemma) catches these.

Fallback

If sentence-transformers isn't installed (e.g. a slim Docker image), this stage falls back to keyword scoring and emits a fallback trace tag.

4

Deterministic tool calls

Seven tools fire on every moderate run, deterministically (no model needed):

  1. lookup_statute(juris, topic) — legal anchoring.
  2. lookup_hotline(country) — locale-aware contact.
  3. check_agency_license(name, juris) — POEA / EAA / MOM lookup.
  4. calculate_max_fee(juris, role) — fee-cap citation.
  5. identify_ilo_indicators(text) — ILO C029 #1–#11 regex match.
  6. lookup_embassy(origin, destination) — consulate + secondary hotline.
  7. search_known_actors(text) — past-violations check; bumps severity.

Why tools instead of just RAG?

Two things RAG can't do: (a) compute ("is USD 5000 over the fee cap?" needs a comparison, not a passage), and (b) cross-reference ("is Pacific Coast Manpower licensed?" needs a registry, not the law). Tools handle the non-text-shaped parts of the answer.

Why deterministic and not Gemma-driven?

For the demo, we want every example to fire the same tools in the same order so judges see consistent, reproducible behaviour. The architecture supports Gemma-emitted tool calls too (see raw_python/gemma4_*_v1.py) but the synchronous demo path is deterministic for clarity.

5

Gemma 4 classification

The final classifier. Receives a structured prompt with: the original input, the heuristic findings (so it can confirm or refute the prescan), the matched KB passages (so it can cite them), an explicit severity-anchor table tied to ILO / Palermo / RA 8042 thresholds, and the verdict rules. Returns a JSON object with verdict ∈ {block, review, pass}, severity 0–10, and a one-sentence reasoning.

Why anchor with the heuristic?

Without the anchor, Gemma 4 occasionally returned severity: 1 for inputs with passport confiscation. Anchoring it to the heuristic prescan + the explicit severity table fixes this to high single-digits consistently. The post-processor takes max(gemma, heuristic) as a safety net.

Why Gemma 4 specifically?

Three Gemma 4 features matter for this rubric:

  • Native multimodal — image + text in one model means the same Gemma can read a contract photo and the accompanying message. We use the text-only path here for speed; multimodal is wired in load_gemma_smart.
  • Native function calling — the chat template accepts a tools list and emits structured tool calls. The pipeline's deterministic tools are an MVP; the full agentic loop is one bool flag away.
  • On-device deployable — E4B in 4-bit fits a Kaggle T4 (15 GB VRAM) and a $300 LiteRT phone. NGOs that can't send case data to the cloud can still run the analysis.

6

Output: verdict + evidence + action

Every moderate run returns a JSON payload that includes:

  • verdict + severity + reasoning
  • matched_signals — heuristic findings as objects
  • kb_hits — every passage retrieved by GREP + RAG
  • tool_calls — every tool invocation with args + result
  • ilo_indicators — which ILO C029 indicators fired
  • fee_cap, license_check, embassy, known_actors
  • hotline — locale-aware contact

This is what makes the verdict actionable: a moderator doesn't just see "BLOCK", they see the statute, the embassy contact, and the unverified-licence flag they need to write the take-down report.
