How to add new evidence (operator workflow)

  1. Redact first. Blur every identifiable face. Remove every full name (replace with [NAME REDACTED] or [OFW]). Cover passport / ID numbers with a solid bar. Cover phone numbers / emails / handles. Use a real image editor — do not rely on browser-side blurs.
  2. Save the redacted PNG/JPG into packages/duecare-llm-server/src/duecare/server/static/evidence/
  3. Add a manifest entry in static/evidence/manifest.json with the schema below. redaction_status must be "REDACTED" — anything else is rejected at build.
  4. Rebuild the wheel and re-upload. The gallery will pick up the new asset on next page load.

Manifest schema

{
  "id": "doxxing_001",
  "kind": "image",
  "filename": "doxxing_001_redacted.png",
  "title": "Bank Hongkong -- 'wanted poster' template",
  "caption": "Predatory lender's public-shaming Facebook post. Names + face blurred. Documents passport-as-collateral pattern.",
  "category": "doxxing",
  "redaction_status": "REDACTED (faces blurred, full name removed)",
  "source_note": "Public FB page, captured Apr 2024"
}
