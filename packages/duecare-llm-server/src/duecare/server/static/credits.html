Duecare stands on a lot of open-source shoulders. Every component listed here is used under its original licence; nothing has been forked or modified beyond standard configuration.

Models — Google Gemma 4 (Apache 2.0)

Built with Google's Gemma 4. Duecare supports four variants depending on the deployment target. Attribution per the Gemma terms of use: "Built with Google's Gemma."

VariantApprox paramsTypical deploymentHF Hub
Gemma 4 E2B-it~2BMobile via LiteRT, ~2GB 4-bit google/gemma-4-e2b-it
Gemma 4 E4B-it~4BDefault consumer GPU, ~5.5GB 4-bit google/gemma-4-e4b-it
Gemma 4 26B-A4B-it~26B (MoE)Single A100 or 2x T4, ~14GB 4-bit google/gemma-4-26b-a4b-it
Gemma 4 31B-it~31B (dense)2x T4 via Unsloth (device_map="balanced"), ~18GB 4-bit google/gemma-4-31b-it

Inference + training stack

  • Unsloth — 4-bit FastModel + the 31B-on-2xT4 path via device_map="balanced". github.com/unslothai/unsloth · LGPL-3.0
  • HuggingFace transformers — model loading, tokenization, generation. github.com/huggingface/transformers · Apache 2.0
  • bitsandbytes — nf4 4-bit quantization. github.com/bitsandbytes-foundation/bitsandbytes · MIT
  • sentence-transformers — RAG retrieval (all-MiniLM-L6-v2). github.com/UKPLab/sentence-transformers · Apache 2.0
  • llama.cpp — GGUF runtime path for desktop / on-device. github.com/ggerganov/llama.cpp · MIT
  • Ollama — local-runtime adapter (scripts/run_local_gemma.py). github.com/ollama/ollama · MIT
  • Google AI Edge LiteRT — mobile runtime path. ai.google.dev/edge/litert · Apache 2.0
  • PyTorch · triton · torchcodec · torchvision · timm — supporting compute
