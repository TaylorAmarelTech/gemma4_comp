# Test before commit — the discipline

> Auto-loaded at the project memory level.

## Before any commit that touches code

Run the relevant duecare test scope:

```bash
# if you touched a single module
duecare test src/forge/agents/judge

# if you touched a whole layer
duecare test src/forge/agents --recursive

# if you touched something core / cross-layer
duecare test src/forge --recursive
```

Every test must pass, or the commit doesn't happen. No exceptions.

## Before any commit that touches meta files

The meta files (PURPOSE.md, AGENTS.md, INPUTS_OUTPUTS.md, HIERARCHY.md,
DIAGRAM.md, TESTS.md, STATUS.md) are auto-generated by
`scripts/generate_forge.py` from the `MODULES` descriptor list. Do not
edit meta files by hand. If you need to change a module's purpose,
inputs, outputs, or dependencies:

1. Edit the descriptor in `scripts/generate_forge.py`.
2. Re-run `python scripts/generate_forge.py`.
3. Verify the regenerated meta files look right.
4. Commit both the generator change AND the regenerated meta files.

## Before any commit that touches the scaffold generator

Also re-run it with the current `MODULES` list to make sure it still
produces valid output across all 58 modules.

## Before any PR

- [ ] `duecare test src/forge --recursive` passes
- [ ] `ruff check src/` passes
- [ ] `mypy src/forge` passes
- [ ] No new files outside the module tree (packages/, src/, scripts/, configs/, docs/, tests/, .claude/, .github/)
- [ ] No PII (see `.claude/rules/10_safety_gate.md`)
- [ ] If a module's contract changed, `HIERARCHY.md` on every dependent module is regenerated
- [ ] Commit message references the module(s) changed by path

## Why this matters

Rule 4 in `00_overarching_goals.md`: "Real, not faked for demo." The
code repo is one of the two sources of truth judges use to verify the
submission. A code repo with broken tests is a signal that the tech
isn't real.

Fix broken tests before they accumulate.
